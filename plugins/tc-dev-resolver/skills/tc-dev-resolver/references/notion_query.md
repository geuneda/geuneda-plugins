# 노션에서 TC 번호로 페이지 찾기

`tc-dev-resolver`는 사용자가 인자로 넘긴 `번호`를 단서로 TC 데이터소스 안에서 한 건의 페이지를 식별해야 한다. 번호는 사용자가 노션에서 직접 본 값을 옮긴 것이라 오타나 부분 입력이 자주 발생한다 — 한 번에 못 찾으면 단계적으로 범위를 넓혀 시도한다.

## 입력 번호 정규화

사용자가 넘긴 번호는 다음과 같이 변형이 있을 수 있다. 검색 직전에 한 번 다듬는다:

| 사용자 입력 | 정규화 후 검색 키 |
|-------------|-------------------|
| `TC-20260520-보스전스킬` | `TC-20260520-보스전스킬` (그대로) |
| `20260520-보스전스킬` | `TC-20260520-보스전스킬` (TC- 접두 보강) |
| `보스전스킬` | `보스전스킬` (부분 일치 검색용) |
| `TC 20260520 보스전스킬` (공백) | `TC-20260520-보스전스킬` (공백 → 하이픈) |
| 끝에 따옴표/공백 | 트림 |

정규화 후의 키는 두 가지 모드로 활용한다:
- **정확 일치**: `번호`가 그대로 같은 페이지
- **부분 일치 폴백**: 정확 일치가 0건일 때 키워드를 단어 단위로 쪼개 다시 검색

## 1차 검색: 정확 일치

데이터소스 안에서 `번호`가 정규화된 키와 같은 페이지를 찾는다.

```
notion-search
  query: "<정규화된 번호>"
  data_source_url: "collection://<data_source_id>"
  query_type: "internal"
```

`notion-search`는 title을 우선 매칭하므로 보통 정확한 번호로 1건이 잡힌다.

## 2차 검색: 부분 일치 폴백

1차에서 0건이면 다음 두 가지 폴백을 차례로 시도한다.

### 2-A. 날짜 토큰 제거

`TC-20260520-보스전스킬`처럼 날짜 토큰이 포함된 경우 사용자가 날짜만 잘못 기억할 수 있다 — `보스전스킬` 부분으로만 다시 검색한다.

```
notion-search
  query: "보스전스킬"
  data_source_url: "collection://<data_source_id>"
  query_type: "internal"
```

### 2-B. 한글 슬러그 일부

여전히 0건이면 슬러그를 더 짧게(앞 4~6자) 잘라 마지막 시도. 그래도 안 잡히면 정말 없는 것으로 본다.

## 결과 처리

### 1건 매칭

페이지를 처리 대상으로 확정한다. 다음을 메모리에 둔다:
- 페이지 URL (`notion-update-page`의 `page_id`로 사용)
- 컬럼 값 전체 (`번호`, `날짜`, `선택`, `우선순위`, `상태`, `내용`, `원인`, `처리 날짜`)

페이지 상세는 `notion-search` 결과에 포함되는 properties로 충분한 경우가 많지만, `내용` 컬럼이 잘려서 오면 `notion-fetch`로 페이지 URL을 한 번 더 조회해 전체 텍스트를 확보한다.

### 2건 이상 매칭

사용자에게 후보 목록을 보여주고 어떤 페이지인지 확인을 받는다. URL과 `내용` 첫 80자, 등록 날짜를 한 번에 같이 보여주는 게 사용자에게 가장 빠르다:

```
입력하신 번호와 일치하는 TC가 여러 건 있습니다. 어떤 페이지를 처리할까요?

1. TC-20260520-보스전스킬 (등록 2026-05-20, 상태: 진행 전)
   [항목] 보스전 - 캐릭터 궁극기 ...
   URL: https://www.notion.so/...

2. TC-20260418-보스전스킬 (등록 2026-04-18, 상태: 완료)
   [항목] 보스전 - 부하 스킬 ...
   URL: https://www.notion.so/...

번호로 답해주세요 (1/2/취소).
```

사용자가 고른 페이지를 처리 대상으로 한다. `취소`면 종료.

### 0건 매칭 (폴백 후에도)

다음 메시지로 종료한다:

```
'{입력 번호}'에 해당하는 TC를 노션에서 찾지 못했습니다.

확인 부탁드립니다:
- 노션 페이지에서 '번호' 컬럼 값을 정확히 복사하셨는지
- 다른 데이터소스(기획, 템플릿)의 페이지가 아닌지

번호를 다시 알려주시면 재시도하겠습니다.
```

사용자가 다른 번호를 주면 처음부터 다시 시작한다.

## 검색 시 주의

- **TC 외 데이터소스로 새지 말 것**: `data_source_url`을 항상 명시한다. 빈 값으로 검색하면 워크스페이스 전역으로 새 나갈 수 있다.
- **query_type=internal**: 외부 검색(웹/구글)이 아닌 노션 내부 검색을 명시한다.
- **한글 검색**: `notion-search`는 한글 토큰을 그대로 처리한다 — 인코딩 변환은 불필요.
- **번호와 내용의 동시 매칭**: 사용자가 번호 대신 `내용`의 키워드("보스전 궁극기")를 줄 때가 있다. `번호`로 0건이면 `내용` 컬럼 검색은 시도하지 않고, "번호 형식으로 알려주세요"라고 한 번 더 안내한다 (잘못된 페이지를 처리하는 것보다 한 번 더 묻는 게 안전).

## 페이지 ID 추출

`notion-update-page`의 `page_id`에는 다음 중 어느 것이든 넣을 수 있다:

- 페이지 URL: `https://www.notion.so/teamsparta/TC-20260520-...-abc123def456`
- 페이지 UUID: `abc123de-f456-...-...-...` (대시 포함)

`notion-search` 결과의 `url` 필드를 그대로 쓰면 가장 안전하다. UUID만 잘라서 쓰려고 정규식으로 추출하지 않는다 — URL 형식이 바뀌어도 그대로 동작한다.
